To set up your project on Vercel, follow the steps bellow:

1. On the Deploy on Vercel page, name your repository and click on **Create**:

![](/docs/images/cleanshot-2026-04-14-at-18.06.03-2x-U5Nz.png)

1. [Create an account](https://outstatic.com/auth/sign-up) or [Sign in](https://outstatic.com/auth/sign-in) to Outstatic with your GitHub account.
2. Create a new project by connecting it to the repository Vercel created on step 1.

![](/docs/images/cleanshot-2026-04-15-at-10.37.14-2x-cxMz.png)

1. In your project page, find **API Keys** on the sidebar and click on it.
2. Copy and paste the link below as your **Callback Origin** (we will change this later). 

```bash
https://localhost:3000/outstatic
```

1. Click **Add Origin**.
2. Click on **Generate API Key.**
3. Copy the API key and paste it into the Deploy on Vercel's **Add Environment Variables** section. Click on Deploy.

![](/docs/images/cleanshot-2026-04-15-at-10.44.02-2x-Y0OD.png)

1. Click on Deploy

When the deploy finishes, open your site. To access the dashboard, add `/outstatic` to your site URL:

Example: `https://myblog.vercel.app/outstatic`

You should see this page:

![](/docs/images/login-screen-I1NT.webp)

1. Now go back to **API Keys** in Outstatic and add your production callback URL: `https://myblog.vercel.app/outstatic`
2. Return to `/outstatic`, and click **Sign in with GitHub** again.

Congratulations! Now you have a website with a full-featured dashboard to edit your content.

To develop your Vercel deployed website locally, please check the [**Local Development**](https://outstatic.com/docs/local-development) page.

We recommend you learn how [**Outstatic manages content**](https://outstatic.com/docs/introduction) and also how to [**fetch content**](https://outstatic.com/docs/fetching-data) from your front end.
